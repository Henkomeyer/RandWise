## Known limitations

- Privacy mode is in-memory only. Persistence can be added later if product requirements call for it.
- Placeholder routes intentionally contain minimal UI and no data loading.
- Dashboard widgets remain fixture-backed until dashboard frontend/backend integration tasks replace them.

## Risks

- Later feature agents should use the shared `MoneyText` component for user-visible monetary values; bypassing it would create privacy-mode gaps.
- The app currently has no icon dependency, so navigation remains text-only.

## Required follow-up

- Dashboard feature work should replace fixture panels with contract-backed widgets while preserving `MoneyText` privacy support.
- Transactions, Add, Budget and More feature agents should replace placeholders without redefining shared primitives.
- QA/accessibility wave should run full viewport and assistive checks once real feature screens land.
