## API or schema decisions
`POST /api/v1/transactions/{id}/categorise` remains backward-compatible and now accepts optional `createRule`, `matchType` and `matchValue` fields. When rule creation is requested, the transaction application service creates a personal category rule and confirms the corrected transaction.

## Known limitations
The frontend does not yet expose a category-correction rule-learning control.

## Risks
Rule conflict handling is deliberately simple: an active conflicting rule for the same normalized match is deactivated before adding the corrected rule.
