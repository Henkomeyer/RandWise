## API or schema decisions
- Public financial-profile enum strings are:
  - `budgetCycleType`: `calendarMonth`, `paydayToPayday`
  - `notificationMode`: `silent`, `confirm`, `coach`
  - `firstDayOfWeek`: lowercase day names, for example `monday`
- `startingBalanceCents` may be negative to represent an overdraft/opening debt position, matching the domain allowance for signed balances.
- `defaultMonthlyIncomeCents`, `safetyBufferCents`, and `savingsCommitmentCents` are non-negative through domain invariants.
- GET returns `404` when the authenticated user has not completed financial onboarding yet.
- PUT returns `200 OK` for both create and update to keep onboarding idempotent.
- No database schema or migration changes were required; the implementation uses the existing `FinancialProfiles` table.
