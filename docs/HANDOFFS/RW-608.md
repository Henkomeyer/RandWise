## API or schema decisions
Added `IncomingMessages.AttemptCount INTEGER NOT NULL DEFAULT 0`. The background processor retries received messages up to three attempts, records retryable failure reasons, and marks exhausted messages failed.

## Known limitations
There is no hosted scheduler loop yet; retry processing is available through the background processor abstraction.

## Risks
Retry timing is immediate in the MVP processor. A later hosted worker should add backoff before production traffic.

## Required follow-up
Add operational scheduling/backoff when the worker is hosted for production.
